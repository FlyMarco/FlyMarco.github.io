AIDL

AIDL 全称为 Android Interface Definition Language，主要用于客户端和服务端通信。

AIDL 支持的数据类型

  1. 基本数据类型
  2. String、CharSequence
  3. List (只有 ArrayList，并且每个元素必须被 AIDL 支持)
  4. Map (只有 HashMap，并且每个元素必须被 AIDL 支持)
  5. Parcelable
  6. AIDL 接口

自定义的 Parcelable 对象都需要定义一个同名的 .aidl 文件，并且声明为 Parcelable 类型。

// Book.aidl
package xx.xx.xx;
parcelable Book;

所有的 Parcelable 和 AIDL 对象都需要显式的 import 进文件。

AIDL中除了基本类型，其他类型都需要标上方向：in、out、inout。in 表示输入型参数，out 表示输出型参数，inout 表示输入输出型参数。

AIDL中只支持方法，不支持声明静态常量，不能使用访问限定符。

// IBookManager.aidl
package xx.xx.xx;

import xx.xx.xx.Book;

interface IBookManager {
    List<Book> getBookList();
    void addBook(in Book book);
}

IBookManager.java 扩展了 IInterface 接口，新增了在 IBookManager.aidl 中声明的方法。

public interface IInterface {
    public IBinder asBinder();
}

静态抽象成员类 Stub 实现了 IBookManager 接口，是服务端的 Binder 对象。

静态成员类 Proxy 实现了 IBookManager 接口，是 Stub 的代理类。

服务端 Service

服务端需要实现 Stub 的抽象方法，具体实现 IBookManager.aidl 中声明的方法，为客户端提供服务。

public class ServerService extends Service {

    private CopyOnWriteArrayList<Book> booklist = new CopyOnWriteArrayList<Book>();

    private IBinder mBinder = new IBookManager.Stub() {

        @Override
        public List<Book> getBookList() throws RemoteException {
            // TODO Auto-generated method stub
            return booklist;
        }

        @Override
        public void addBook(Book book) throws RemoteException {
            // TODO Auto-generated method stub
            booklist.add(book);
        }
    };

    @Override
    public void onCreate() {
        super.onCreate();
        booklist.add(new Book(1,"first"));
        booklist.add(new Book(2,"secound"));
    };

    @Override
    public IBinder onBind(Intent intent) {
        // TODO Auto-generated method stub
        return mBinder;
    }

}

小结
  1. 创建 .aidl 文件，声明方法。自定义 Parcelable 对象需要创建同名的 aidl 文件并且声明，还需要显式引用。
  2. 根据 aidl 文件生成同名 java 文件
    1. 该文件扩展了 IInterface 接口，创建了 aidl 文件中声明的方法
    2. 静态抽象成员类 Stub 继承 Binder 实现这个接口
      1. DESCRIPTOR：Binder 的唯一标识，通常用当前类名表示
      2. asInterface()：将服务端 Binder 对象转化成客户端 AIDL 类型的对象。如果处于同进程，返回 Stub，否则返回 Stub.proxy 对象
      3. asBinder：返回当前 Binder 对象
      4. onTransact(int code, Parcel data, Parcel reply, int flags)：具体执行客户端的请求。code 表明具体执行那个方法，data 为接收到的参数，reply 为返回的参数。如果返回 false 表明客户端请求失败。
    3. Proxy 是 Stub 的代理类，如果不处于同一进程，客户端拿到的 IBinder 对象就是 Proxy
  3. 服务端创建 Service，创建 Stub 的实现对象 mBinder，在 onBind() 方法中返回这个 mBinder 对象
  4. 客户端绑定这个服务，根据返回的 IBinder 对象生成 AIDL 类型的对象，调用其方法
  5. 客户端发起远程请求时，当前线程挂起知道服务端返回数据。
  6. 服务端的 Binder 方法运行在 Binder 线程池中。

RemoteCallbackList

RemoteCallbackList 是专门用于删除跨进程 listener 的接口。当客户端进程终止后，能够自动移除注册的 listener。其内部自动实现了线程同步功能。

RemoteCallbackList 接受 IInterface 类型的参数：

public class RemoteCallbackList<E extends IInterface> {}

其内部结构为一个 ArrayMap：

ArrayMap<IBinder, Callback> mCallbacks = new ArrayMap<IBinder, Callback>();

key 和 value 获取方法：

IBinder binder = callback.asBinder();
Callback cb = new Callback(callback, cookie);

Callback 是局部类，绑定了 listener 和 DeathRecipient：

private final class Callback implements IBinder.DeathRecipient {
    final E mCallback;
    final Object mCookie;

    Callback(E callback, Object cookie) {
        mCallback = callback;
        mCookie = cookie;
    }

    public void binderDied() {
        synchronized (mCallbacks) {
            mCallbacks.remove(mCallback.asBinder());
        }
        onCallbackDied(mCallback, mCookie);
    }
}

在注册时将 Callback 实例添加进 ArrayMap，并且设置死亡代理：

public boolean register(E callback, Object cookie) {
    synchronized (mCallbacks) {
        if (mKilled) {
            return false;
        }
        IBinder binder = callback.asBinder();
        try {
            Callback cb = new Callback(callback, cookie);
            binder.linkToDeath(cb, 0);
            mCallbacks.put(binder, cb);
            return true;
        } catch (RemoteException e) {
            return false;
        }
    }
}

解除注册时删除 callback,并且取消死亡代理

public boolean unregister(E callback) {
    synchronized (mCallbacks) {
        Callback cb = mCallbacks.remove(callback.asBinder());
        if (cb != null) {
            cb.mCallback.asBinder().unlinkToDeath(cb, 0);
            return true;
        }
        return false;
    }
}
