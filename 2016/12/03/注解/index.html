发表于 2016-12-03 | 分类于 Java

注解

注解(也被称为元数据)为我们在代码中添加信息提供一种形式化方法,使我们在稍后某个时刻非常方便地使用这些数据.

Java SE5内置注解有：

Override:当前方法覆盖超类的方法.
Deprecated:被替代的，编译器会发出警告.
SuppressWarnings:关闭编译器警告信息.

自定义注解

Java使用元注解来注解其他的自定义注解.

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Document
@Inherited
public @Interface UseCase {
    public int id();
    public String description() default "description"
}

元注解

  • @Target 表示该注解用于什么地方.

    ElementType.CONSTRUCTOR:构造器的声明
ElementType.FIELD:域声明，包括enum实例
ElementType.LOCAL_VARIABLE:局部变量声明
ElementType.METHOD:方法声明
ElementType.PACKAGE:包声明
ElementType.PARAMETER:参数声明
ElementType.TYPE:类、接口、enum声明，包括注解类型

  • @Retention 表示在什么级别保留注解信息.

    RetentionPolicy.SOURCE:源代码级别，将被编译器丢弃
RetentionPolicy.CLASS:class文件可用，将被VM丢弃
RetentionPolicy.RUNTIME:VM运行期也保留注解，可通过反射机制读取注解信息

  • @Documented 将此注解包含在Javadoc中.

  • @Inherited 允许子类继承父类的注解.作用在类上.

注解元素

注解元素可作用的类型有：

  • 基本类型
  • String
  • Class
  • enum
  • Annotation
  • 上述类型的数组

默认值限制

  • 注解元素不能有不确定的值，即必须具有默认值或者在使用注解时提供元素的值。
  • 非基本类型的元素的值不能为null

补充

  • 没有元素的注解称为标记注解
  • 注解的元素在使用时表现为名-值对的形式，置于注解声明后的括号内
  • 如果注解元素只有一个且命名为value，可以直接给出值而不用名-值对的形式
  • 注解不支持继承

注解处理器

简单的注解处理器一般包括以下几步：

  1. 加载一个类，检查是否带有XX注解
  2. 读取这个类的所有域，获得域上所有注解的数组
  3. 判断每个注解的具体类型，进行相应处理
