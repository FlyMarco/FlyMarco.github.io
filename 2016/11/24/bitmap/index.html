Bitmap简介

发表于 2016-11-24 | 分类于 Android

Bitmap

BMP是一种图形文件格式，将图像定义为由像素点组成。在Windows3.0以前BMP文件格式与显示设备有关，称为设备相关位图DDB，Windows3.0以后BMP文件格式与显示设备无关，称为设备无关位图DIB。

BMP位图文件默认扩展名为BMP或者bmp。

BMP格式结构

BMP文件数据可以分成四部分，按顺序排列为：

  • 位图文件头：标注文件格式、大小等信息
  • 位图信息头：标注图像尺寸、颜色格式等信息
  • 调色板：用以索引对应的颜色的映射表，24位和32位位图无调色板
  • 位图数据：图像数据区

文件信息头

文件类型         //2字节,'BM'
文件大小        //4字节
保留位             //2字节,设置为0
保留位             //2字节,设置为0
数据偏移量        //4字节,大小为文件头、信息头和调色板的总和

位图信息头

信息头大小        //4字节,大小为40字节
图像宽度        //4字节,以像素为单位
图像高度        //4字节，以像素为单位
平面数             //2字节，恒为1
像素位数        //2字节，1、4、8、16、24、32
压缩信息        //4字节，一般为BI_RGB不压缩
数据大小        //4字节
水平分辨率        //4字节，每米像素个数
垂直分辨率        //4字节，每米像素个数
颜色数             //4字节
重要颜色数        //4字节

调色板

蓝色分量        //1字节
绿色分量        //1字节
红色分量        //1字节
保留值            //1字节

Bitmap编码实战

相机数据生成24位Bitmap图像

通过相机预览得到原始NV21数据，将NV21转为RGB格式，将文件头、信息头、图像数据打包写到文件中。

  1. 数据来源PreviewCallback

  2. 数据格式转换NV21到RGB888

    NV21数据格式为：

    YYYYYYYY YYYYYYYY YYYYYYYY YYYYYYYY VUVUVUVU VUVUVUVU

    相邻四个Y分量共用一个VU分量，所以同时取四个Y分量和一个VU分量转换为四个RGB值,转换公式为：

    r = y + 1.402f * u; g = y - 0.344f * v + 0.714f * u;b = y + 1.772f * v;

  3. 数据整合

    完成文件头、信息头的数据填充，与RGB数据整合成BMP文件的所有二进制数据。

    由于BMP要求图像宽度需要是4的整数倍，不足的需要添加字节补齐，补齐后图像实际宽度计算公式为：

    (((width) * (BitCount) / 8) + 3) & ~3

    BMP图像扫描是从下往上的，所以图像数据在排列时先排列最后一行数据，依次存放到首行为止，所以整合数据时保证RGB数据的顺序。

  4. 保存到文件

    通过Java I/O输出到文件。

关键代码

通过计算得到的数据的大小i，得到其在数据信息中的4个字节的存储格式。

void writeArray(String string, byte[] out, int i) {
    if (string.length() == 0) {
        return;
    } else if (string.length() < 2) {
        out[i] = Integer.valueOf(string, 16).byteValue();
    } else {
        out[i] = Integer.valueOf(
                string.substring(string.length() - 2, string.length()), 16)
                .byteValue();
        writeArray(string.substring(0, string.length() - 2), out, i + 1);
    }
}

以下是NV21转为RGB的代码，四个Y分量对应各一个V、U分量，转为四个RGB像素值。

for (int i = 0, k = 0; i < size; i += 2, k += 2) {
    y1 = data[i] & 0xff;
    y2 = data[i + 1] & 0xff;
    y3 = data[width + i] & 0xff;
    y4 = data[width + i + 1] & 0xff;

    v = data[offset + k] & 0xff;
    u = data[offset + k + 1] & 0xff;
    u = u - 128;
    v = v - 128;

    pixels[i] = convertYUVtoARGB(y1, u, v);
    pixels[i + 1] = convertYUVtoARGB(y2, u, v);
    pixels[width + i] = convertYUVtoARGB(y3, u, v);
    pixels[width + i + 1] = convertYUVtoARGB(y4, u, v);

    if (i != 0 && (i + 2) % width == 0)
        i += width;
}

private int convertYUVtoARGB(int y, int u, int v) {
    int r, g, b;

    r = y + (int) 1.402f * u;
    g = y - (int) (0.344f * v + 0.714f * u);
    b = y + (int) 1.772f * v;
    r = r > 255 ? 255 : r < 0 ? 0 : r;
    g = g > 255 ? 255 : g < 0 ? 0 : g;
    b = b > 255 ? 255 : b < 0 ? 0 : b;
    return 0xff000000 | (r << 16) | (g << 8) | b;
}

自定义16色Bitmap图像

16色Bitmap图像中每两个像素点占有1个字节，总共含有16色的调色板，数据区存储的是指向调色板的索引值，最终颜色由调色板定义。

注意

  1. 此时Bitmap中的文件大小和数据偏移量都要加上调色板的数据大小64B
  2. 像素位数为4，即一个字节表示两个像素
  3. 所使用颜色数为16

调色板的定义如下：

灰度图

那白色和黑色之间按对数关系分为若干等级称为灰度，灰度分为256阶，用灰度表示的图像称为灰度图。

将RGB转换为灰度的方式有：

  1. 浮点算法：Gray=Rx0.299+Gx0.587+Bx0.114
  2. 整数方法：Gray=(Rx299+Gx587+Bx114+500)/1000
  3. 移位方法：Gray=(Rx76+Gx151+Bx28)>>8
  4. 平均值法：Gray=(R+G+B)/3

编码实例

本例用24位彩色图转为8位灰度图，在灰度图中每个像素占有1B，并且带有1024B的调色板，其中每个调色板索引值中R、G、B分量都相等。

定义调色板：

COLOR_LIST = new byte[256 * 4];
for (int i = 0; i < 256; i++) {
    COLOR_LIST[i * 4] = (byte) i;
    COLOR_LIST[i * 4 + 1] = (byte) i;
    COLOR_LIST[i * 4 + 2] = (byte) i;
    COLOR_LIST[i * 4 + 3] = (byte) 0xFF;
}
